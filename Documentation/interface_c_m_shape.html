- (void) setCollisionType: (cpCollisionType)  type  

A user defined collision type which can be used to identify a type of object on screen.

(is used by the collision detection)

For example, if a have a ball and a block on the screen i could identify the ball as type 1 and the block as type 2.

- (void) setElasticity: (float)  elasticity  

Elasticity of the shape.

A value of 0.0 gives no bounce, while a value of 1.0 will give a ìperfectî bounce. However due to inaccuracies in the simulation using 1.0 or greater is not recommended however.

Parameters:
elasticity a value between 0.0 and (adviced) 1.0
- (void) setFriction: (float)  friction  

Friction coefficient.

Chipmunk uses the Coulomb friction model, a value of 0.0 is frictionless.

For a listing of various frictions see: http://www.roymech.co.uk/Useful_Tables/Tribology/co_of_frict.htm

Parameters:
friction a value 0f 0.0 is frictionless
- (void) setGroup: (cpGroup)  group  

Shapes in the same non-zero group do not generate collisions.

Useful when creating an object out of many shapes that you don’t want to self collide. Defaults to CP_NO_GROUP.

Parameters:
group the group
- (void) setLayer: (cpLayers)  layer  

Shapes only collide if they are in the same bit-planes.

i.e. (a->layers & b->layers) != 0 By default, a shape occupies all bit-planes.

Wikipedia has a nice article on bitmasks if you are unfamiliar with how to use them. Defaults to CP_ALL_LAYERS.

http://en.wikipedia.org/wiki/Mask_(computing)top

layer the layer
